# 🎬 混剪工坊

网页版视频自动混剪工具。把**背景音乐**、**人声配音**、**视频片段**（一次最多 100 条）丢进去，自动混剪成片。

全部在本地运行，素材不上传到任何外部服务器。

> **打包版**自带便携 Node + FFmpeg + whisper，双击 `start.bat` 即用。
> **从 GitHub 拉的源码版**为了体积，没有包含运行时和媒体文件，请按下面「从源码运行」配置。

## 从源码运行（开发者）

本仓库**不含**便携运行时（`runtime/`，约 553MB）、依赖（`node_modules/`）、生成的视频与日志。自己准备：

1. **Node.js 24+**、**FFmpeg**（命令行可用），可选 **whisper.cpp**（自动字幕用，把 `whisper-cli.exe` + `ggml-base.bin` 放到 `runtime/whisper/`；不配则字幕功能关闭）。
   - 若把它们放进 `runtime/node`、`runtime/ffmpeg/bin`、`runtime/whisper`，则与打包版结构一致、`start.bat` 直接可用；也可改用系统已装的 Node/FFmpeg。
2. `npm install` 安装依赖（express、multer 等，见 `package.json`）。
3. `node server.js` 启动，浏览器开 `http://localhost:5173`。
4. **数据与密钥**在应用目录外的数据目录（默认同级的 `..-data`，即 `D:\Dvideo-tool-data`），仓库不含；按需创建：
   - `llm_config.json`：`{ "base_url", "api_key", "model" }`（OpenAI 兼容接口，用于「一键生成脚本」/广告数据分析）。
   - `minimax_config.json`：MiniMax T2A 配音的 `{ "api_key", "base_url", "voice_id", ... }`。
   - TikTok 投放数据/上传：`D:\tkads_export\config.json`（`access_token` + `advertiser_ids`），路径可用环境变量 `TT_CONFIG` 覆盖。
   - **以上含密钥，切勿提交到仓库**（已在 `.gitignore` 排除）。

### 主要功能
混剪成片（多条 / 字幕 / 音乐闪避）· 背景音乐库 / 配音库 / 视频素材库（文件夹分类）· 脚本库（按 SKU 分类）· **配音工坊**（MiniMax 文本转语音，303 音色 / 试听）· 混剪设置**模版** · **一键生成并上传**（按 SKU 选资源 → 批量出片 → 传到 TikTok 广告户）· TikTok 投放数据（按视频聚合，含 2 秒完播率，只看转化类）· **🪄 一键生成脚本**（结合投放数据 + 大模型）。

## 使用方法

1. 双击 **`start.bat`**，会自动打开浏览器 `http://localhost:5173`。
2. 在网页里：
   - **上传素材**：拖入视频片段、背景音乐、人声配音。
   - **混剪设置**：选画面方向、时长基准、片段顺序、混音方式等。
   - 点 **开始混剪**，等待进度走完。
3. 成片可在线预览，并点 **下载成片**。

关闭命令行黑窗口即停止服务。

## 混剪逻辑

1. 探测配音/音乐时长，确定成片总时长 **T**。
2. 把所有视频片段统一到同一画布、帧率，拼成无声「蒙太奇」。
3. （可选）根据**人声配音**用本地语音识别生成字幕，时间轴自动对齐语音。
4. 蒙太奇循环或裁切到 T，叠加音轨（并烧录字幕）：
   - **人声为主，音乐自动闪避**：有人声说话时背景音乐自动压低（ducking）。
   - **固定音量比例**：配音与音乐各按设定音量混合。
5. 输出 H.264 + AAC 的 MP4（`output/` 目录下）。

## 自动字幕

勾选「自动字幕」后，工具会用内置的本地语音识别引擎（whisper.cpp + base 模型，
全程离线、不上传）把**人声配音**转成带时间轴的字幕，并烧录到画面底部，
字幕出现/消失的时间与语音自动对齐。成片完成后还可单独下载 `.srt` 字幕文件。

- 需要上传人声配音才能生成字幕。
- 中英文自动识别；中文字体使用「微软雅黑」。
- 配音越长，识别耗时越久（一般十几秒到一两分钟）。
- 想要更高识别精度，可把更大的模型（如 `ggml-small.bin` / `ggml-medium.bin`，
  从 huggingface.co/ggerganov/whisper.cpp 下载）放到 `runtime/whisper/models/`，
  并在 `server.js` 顶部把 `WHISPER_MODEL` 指向它。

## 素材库（背景音乐 / 配音）

背景音乐和人声配音可以保存进**素材库**，下次直接选用，不必重复上传：

- 上传新的背景音乐/配音时，勾选其下方的「**把这次上传的存入库**」，本次上传就会被保存。
- 之后在「背景音乐库 / 配音库」里会出现对应的小标签，**点一下即可选用**；再点一下取消。
- 标签上的 ✕ 可从库中删除。
- 选了库里的项后就无需再上传该音频；也可随时改成上传新文件（二选一）。

保存的文件位于 `library/bgm/` 和 `library/voice/`。

## 脚本库

页面**最顶部**的「📝 脚本库」用来保存你的口播脚本 / 文案，随时取用：

- **批量导入（多段自动分段）**：一次粘贴多段脚本，选「分段方式」（自动识别 /
  空行 / 分隔线 --- / 编号 1. 2. / 第1条 / 不拆分），点「识别分段」预览，
  每段标题可改、可删，确认后「全部保存」。自动识别优先级：分隔线 > 编号 > 空行。
- **单段保存**：直接粘贴一段，点「保存」。
- **上传 .txt**：选文件后自动识别分段并预览。
- 列表里点标题/摘要可**展开全文**，点「编辑」改名/改内容，点「复制」复制，点「删除」移除。每页显示 5 条，可翻页。

### 🪄 一键生成脚本（结合投放数据 + AI）

点「🪄 一键生成」：把脚本库的脚本按名字与 **TikTok 投放视频**一一对应（广告名里包含脚本名），
按真实表现（ROAS/转化/CTR/花费）排序，喂给大模型，**生成一条全新的口播脚本**，
填入编辑框供你润色后保存；下方显示参考依据（各脚本的投放数据）。

- 大模型配置在外部数据目录 `D:\Dvideo-tool-data\llm_config.json`（`base_url` / `api_key` / `model`，
  OpenAI 兼容接口；也可用环境变量 `LLM_BASE_URL` / `LLM_API_KEY` / `LLM_MODEL` 覆盖）。当前用 SKG 内部 Qwen（qwen-plus）。
- 若未配置大模型，则退化为**离线拼装**：把高表现脚本的开头/主体/结尾拼成草稿。

脚本以 JSON 形式保存，跨次使用不丢失。

## 数据安全（升级不丢数据）

所有需要长期保留的数据 —— **背景音乐库、配音库、脚本库** —— 都保存在
**应用文件夹外部**的独立数据目录：

```
D:\Dvideo-tool-data\
  library\bgm\      背景音乐库
  library\voice\    配音库
  scripts\          脚本库
```

因此**升级、替换、清空 `D:\Dvideo-tool` 都不会丢失你的素材库和脚本库**。
首次升级到此版本时，旧版本存在应用内 (`library/`、`scripts_lib/`) 的数据会
**自动迁移**到外部数据目录。

- 想换数据目录：启动前设置环境变量 `MIXER_DATA_DIR` 指向你要的路径。
- 想备份：直接拷贝整个 `D:\Dvideo-tool-data` 文件夹即可。
- `uploads/`（临时素材）和 `output/`（成片）属于工作目录，可随时清理。

## TikTok 投放数据

页面底部「📊 TikTok 投放数据」直接对接 TikTok Marketing API，按**投放的视频**
拉回花费、曝光、点击、CTR、转化、CPA、GMV、ROAS。

- 凭证复用 `D:\tkads_export\config.json`（`access_token` + `advertiser_ids`）。
  **Token 只在服务端使用，不会发送到浏览器。** 可用环境变量 `TT_CONFIG` 指定其它配置路径。
- 选好日期区间，点「拉取数据」；结果按花费排序，可搜索视频名 / video_id。
- 原理：拉全部广告得到 `ad_id → video_id / ad_name` 映射，再拉广告级报表（按天），
  按 `video_id` 聚合得到每个视频的整体表现。结果服务端缓存 5 分钟。

> 若提示鉴权失败（code 40100），多半是 Token 过期，需在 TikTok 后台重新授权后更新 config.json。

## 运行日志

服务运行过程会记录到根目录的 **`server.log`**（启动、每次上传、混剪参数、
字幕生成、成片输出，均带时间戳）。也可在浏览器打开
`http://localhost:5173/api/logs` 查看最近 500 行日志。

## 设置说明

| 设置 | 说明 |
|------|------|
| 画面方向 | 竖屏 1080×1920 / 横屏 1920×1080 / 方形 1080×1080 |
| 帧率 | 24 / 30 / 60 fps |
| 时长基准 | 成片长度对齐到 配音 / 音乐 / 片段总长 |
| 片段顺序 | 按文件名 / 按上传顺序 / 随机打乱 |
| 一次生成条数 | 1~5 条；多条时每条用不同片段顺序，得到不同版本（共用同一配音与字幕） |
| 每段截取秒数 | 0=用整段；设为 2~3 秒可做快节奏卡点感 |
| 混音方式 | 自动闪避 / 固定比例 |
| 自动字幕 | 根据人声配音生成并烧录字幕，时间轴对齐语音 |

## 目录结构

```
Dvideo-tool/
  start.bat          一键启动
  server.js          后端服务
  public/            网页前端
  runtime/node       便携版 Node.js
  runtime/ffmpeg     便携版 FFmpeg
  runtime/whisper    便携版语音识别（whisper.cpp + base 模型）
  uploads/           上传的素材（按任务分目录）
  output/            生成的成片
  server.log         运行日志
```

## 端口被占用？

默认端口 5173。如需更换，启动前设置环境变量，例如 PowerShell：

```powershell
$env:PORT=8080; .\runtime\node\node.exe server.js
```
